# SmartQA 项目启动使用文档

本文档用于本机开发、联调和验收。当前系统只保留 SmartQA P0 业务：老板端和客服端共用同一个登录入口，登录后按角色自动进入不同菜单和数据范围。界面只体现老板端/客服端两套身份视图；内部 `tenant_id` 仅用于底座兼容和数据隔离，不作为用户可操作功能。

## 7. AI 质检命令

调用阿里模型执行真实质检：

```powershell
cd F:\Project\SmartQAServiceCloud\backend
python scripts\smartqa_pipeline.py run-ai --limit 1
```

已用真实阿里 `qwen3.7-plus` 跑通过每日抽检链路。新版规则要求同时输出客服质检和客户意向结构，已完成 53 条真实会话结构化结果，覆盖全部 18 个客服。

常规每日抽检：

```powershell
python scripts\smartqa_pipeline.py daily-qc --limit 100 --execute
```

如果只是做验收或小批量补跑，推荐使用短会话样本，仍然是真实千牛聊天和真实阿里模型：

```powershell
python scripts\smartqa_pipeline.py daily-qc-short --limit 40 --max-messages 20 --execute
```

指定某些客服补跑：

```powershell
python scripts\smartqa_pipeline.py qc-staff-short 13 17 --per-staff 1 --max-messages 20 --execute
```

如果模型命令被中断，先恢复运行中任务：

```powershell
python scripts\smartqa_pipeline.py reset-running --all
```

全量任务耗时较长，建议在服务器后台执行并观察 `model_call_log`、`qc_task` 与 `qc_result` 增长。

## 8. 自动同步和每日质检

后端启动后会按配置启用 SmartQA 定时任务：

- 每日 07:30、12:30、20:30 自动同步源库 `aizhijian` 到业务库 `smartqa`，并重建 ODS/DIM/DWD 数据。
- 每日 23:00 自动创建并执行每日抽检，默认抽检 100 个会话。
- 每日抽检优先保证每个客服至少有一个会话进入质检，再按最近会话补足到 100 个。
- 老板端“每日数据批次”页面提供“立即同步”，对应接口：`POST /api/v1/smartqa/sync/source-db`。
- 老板端“AI分析任务”页面提供“今日抽检”和“抽检并执行”，对应接口：`POST /api/v1/smartqa/qc/tasks/daily-sample`。

查看调度状态：

```powershell
cd F:\Project\SmartQAServiceCloud\backend
@'
from app.core.ap_scheduler import SchedulerUtil
print(SchedulerUtil.get_jobs())
'@ | python -
```

## 11. 停止服务

如果是在 PowerShell 窗口里启动，直接在对应窗口按 `Ctrl + C`。

如果是后台进程方式启动，先查端口进程：

```powershell
Get-NetTCPConnection -State Listen | Where-Object { $_.LocalPort -in 8001,5180 } | Select-Object LocalAddress,LocalPort,OwningProcess
```

再停止指定进程：

```powershell
Stop-Process -Id 进程ID
```
