# SmartQA 项目启动使用文档

本文档用于本机开发、联调和验收。当前系统只保留 SmartQA P0 业务：老板端和客服端共用同一个登录入口，登录后按角色自动进入不同菜单和数据范围。界面只体现老板端/客服端两套身份视图；内部 `tenant_id` 仅用于底座兼容和数据隔离，不作为用户可操作功能。

## 8. 自动同步和每日质检

后端启动后会按配置启用 SmartQA 定时任务：

- 每日 07:30、12:30、20:30 自动同步源库 `aizhijian` 到业务库 `smartqa`，并重建 ODS/DIM/DWD 数据。
- 每日 23:00 自动创建并执行每日抽检，默认抽检 100 个会话。
- 每日抽检优先保证每个客服至少有一个会话进入质检，再按最近会话补足到 100 个。
- 老板端“千牛数据源”页面提供“立即同步”，对应接口：`POST /api/v1/smartqa/sync/db`。
- 老板端“AI质检任务”页面提供“今日抽检”和“抽检并执行”，对应接口：`POST /api/v1/smartqa/qc/tasks/daily-sample`。

查看调度状态：

```powershell
cd F:\Project\SmartQAServiceCloud\backend
@'
from app.core.ap_scheduler import SchedulerUtil
print(SchedulerUtil.get_jobs())
'@ | python -
```

接口联调建议用 CLI/API 方式，不通过浏览器自动化：

- 老板登录返回 token，并加载老板端菜单和全量数据范围。
- 客服登录返回 token，并只返回客服端菜单。
- `/api/v1/platform/menu/*`、系统用户/角色/部门/岗位管理等模板维护入口不作为 P0 对外接口。
- 老板接口可访问 `/api/v1/smartqa/staff-users`、`/api/v1/smartqa/qianniu/summary`、`/api/v1/smartqa/qc/tasks`。
- 客服访问上述老板接口返回 403。
- 老板/客服都能访问各自有权限的 SmartQA 看板、会话、质检结果接口。

## 11. 停止服务

如果是在 PowerShell 窗口里启动，直接在对应窗口按 `Ctrl + C`。

如果是后台进程方式启动，先查端口进程：

```powershell
Get-NetTCPConnection -State Listen | Where-Object { $_.LocalPort -in 8001,5180 } | Select-Object LocalAddress,LocalPort,OwningProcess
```

再停止指定进程：

```powershell
Stop-Process -Id 进程ID
```
