# SmartQA 客服质检与客户意向 BI 开发规则

> 项目：SmartQA Service Cloud / AI 客服质检系统  
> 版本：P0.2  
> 生成日期：2026-06-27  
> 用途：作为后续开发“唯一规则依据”，覆盖客服质检、客户意向度、老板端 BI、客服端明细、AI 输出结构和落库字段。  
> 数据场景：淘宝千牛获客，后续对接先发微信承接转化客户。

## 1. 核心结论

SmartQA 不能只做“客服有没有扣分”，也不能只给客服一个“客户有意向”的抽象标签。系统必须同时回答两类问题：

| 对象 | 必须回答的问题 | 开发输出 |
|---|---|---|
| 老板 | 整体服务质量怎么样、哪些客服漏了高意向客户、留资和先发承接哪里断、哪些商品机会多 | 老板总览、高意向客户池、客服表现、商机漏斗、问题分布、商品机会 |
| 客服 | 我名下哪些客户有意向、为什么有意向、现在缺什么、我哪里没做好、下次怎么说 | 我的意向客户、客户详情、我的质检问题、我的改进建议、我的跟进任务 |

开发时必须坚持：

| 原则 | 说明 |
|---|---|
| 意向必须落到客户 | 客服端必须展示具体客户，不允许只展示“高意向 5 个” |
| 意向必须有原因 | 说明客户为什么有意向：需求、规格、数量、预算、时间、联系方式、下单信号 |
| 判断必须有证据 | 高意向、扣分、留资、拒绝等关键判断必须关联消息 ID |
| 客服必须能行动 | 每个高意向客户都要给下一步动作和建议话术 |
| 老板看全局，客服看本人 | 老板看全部客户和客服排行；客服只能看自己负责的客户、问题和建议 |

## 2. 关键业务口径：联系方式不是违规项

本项目是淘宝千牛获客场景。客服在合适时机询问客户微信、电话、手机号或其他联系方式，是允许且正向的业务动作，用于后续先发微信承接。

| 场景 | 质检判断 | 意向判断 | 说明 |
|---|---|---|---|
| 客户需求明确后，客服询问微信/电话便于发方案、报价、案例 | 正向，不扣分 | 加分 | 属于转化推进 |
| 客户咨询定制、大件、批量、工程，客服引导留联系方式 | 正向，不扣分 | 加分 | 属于必要承接 |
| 客户主动问能否加微信或电话沟通 | 强正向 | 强意向 | 需要记录并进入承接 |
| 客服先回答核心问题，再自然询问联系方式 | 正向 | 加分 | 推荐方式 |
| 客服完全不回答问题，只反复要微信/电话 | 扣分 | 不加分 | 问题解决不足、体验差 |
| 客服强迫、威胁、催促客户留资 | 扣分 | 风险 | 服务态度问题 |
| 客服获取联系方式后不确认、不记录、不交接 | 扣分 | 留资断点 | 影响先发承接 |

开发和 Prompt 中必须写清：

```text
先服务，再承接；能留资，是加分；只留资不服务，才扣分。
不要把客服询问微信/电话/手机号判为违规。
```

## 3. 数据基础与分析粒度

| 数据 | 表/字段 | 用途 |
|---|---|---|
| 千牛会话 | `dwd_qn_conversation` | 会话级质检、客户意向判断、页面筛选 |
| 千牛消息 | `dwd_qn_message` | 原文证据、响应时间、话术分析 |
| 客服 | `dim_staff`、`dim_staff_account` | 客服表现、客服端权限过滤 |
| 客户 | `dim_customer`、`dim_customer_identity` | 客户意向、客户明细、后续身份打通 |
| 店铺 | `dim_shop` | 店铺 BI |
| 商品 | `dim_product` | 商品机会、商品问题分析 |
| 质检 | `qc_result`、`qc_issue`、`qc_issue_evidence` | 服务分、问题、证据、建议话术 |
| 先发预留 | `ods_xianfa_converted_customer`、`map_qn_xianfa_customer` | 微信承接、转化匹配 |

| 分析粒度 | 主要用途 |
|---|---|
| 消息 | 证据、响应间隔、关键词、情绪变化 |
| 会话 | 质检分、意向分、留资状态、转化阶段 |
| 客户 | 多次咨询累计意向、历史最高意向、沉默与复访 |
| 客服 | 服务质量、问题分布、高意向客户承接能力 |
| 商品 | 高意向商品、定制/批量需求、价格异议 |
| 店铺 | 店铺整体服务质量、客户意向结构 |
| 日期 | 趋势、环比、每日异常 |
| 规则版本 | 规则调整前后对比 |

## 4. 双评分体系

每个会话必须同时输出两个分，不要混在一个分里。

| 分数 | 满分 | 评价对象 | 用途 |
|---|---:|---|---|
| `staff_quality_score` | 100 | 客服 | 服务质量、扣分、培训、客服表现 |
| `customer_intent_score` | 100 | 客户 | 意向等级、跟进优先级、商机漏斗 |

| 组合 | 解释 | 页面处理 |
|---|---|---|
| 客户意向高 + 客服分高 | 高价值且接待好 | 进入优秀案例、继续推进 |
| 客户意向高 + 客服分低 | 高价值风险会话 | 老板预警、客服待改进、优先跟进 |
| 客户意向低 + 客服分高 | 正常低价值会话 | 不重点跟进，保留服务样本 |
| 客户意向低 + 客服分低 | 低价值但服务有问题 | 只做客服培训，不作为商机重点 |

## 5. 客服质检维度

客服质检满分 100 分。

| 编码 | 维度 | 权重 | 判断重点 | 典型问题 | 客服改进/建议话术方向 | BI 字段 |
|---|---|---:|---|---|---|---|
| SQ01 | 响应及时性 | 12 | 首响、平均响应、关键问题是否超时 | 长时间未回复、连续忽略 | `我先帮您确认一下，稍等马上回复您。` | `first_response_seconds`、`avg_response_seconds`、`timeout_count` |
| SQ02 | 接待完整度 | 8 | 开场、承接、收尾、下一步 | 只发模板、客户问完就断 | 明确下一步：报价、发链接、留资、跟进 | `reception_complete` |
| SQ03 | 需求识别与追问 | 14 | 是否补齐商品、场景、尺寸、数量、预算、时间 | 客户要定制但没问尺寸/数量 | `您是阳台还是庭院使用？方便发下尺寸吗？` | `missing_infos`、`need_probe_missing` |
| SQ04 | 商品与方案专业度 | 14 | 规格、材质、承重、物流、定制、替代方案 | 回答含糊、只发链接不解释 | 给规格差异、适用场景和替代方案 | `professional_score` |
| SQ05 | 问题解决完成度 | 14 | 客户核心问题是否正面回答 | 问价格/尺寸/物流未答到点 | 先回答问题，再推进下一步 | `question_answer_rate`、`unanswered_questions` |
| SQ06 | 转化推进与留资承接 | 16 | 报价、下单引导、联系方式、先发承接 | 高意向未报价、未留资、留资后未交接 | `方便留个微信/电话吗？我整理方案和报价发您。` | `quote_given`、`order_guided`、`contact_requested` |
| SQ07 | 情绪态度与沟通温度 | 8 | 礼貌、耐心、自然，不冷漠不攻击 | 短硬回复、敷衍、不耐烦 | 增加承接和解释：`可以的，我帮您看一下。` | `attitude_issue_count`、`sentiment_risk` |
| SQ08 | 异议处理能力 | 6 | 贵、再看看、不放心、比价时是否有效处理 | 客户说贵后沉默或硬怼 | 先共情，再解释，再给选择 | `objection_type`、`objection_handled` |
| SQ09 | 合规与风险控制 | 6 | 虚假承诺、绝对化表达、隐私过采、冲突 | 保证效果/时效、要无关敏感信息 | 用条件式表达：`我先确认库存/工期后给您准确答复。` | `compliance_risk_count`、`risk_level` |
| SQ10 | 数据记录与交接 | 2 | 联系方式、需求、下一步是否记录 | 客户给联系方式后没确认、没承接 | `收到，我记录一下，稍后同步方案和报价。` | `handoff_confirmed`、`handoff_missing` |

### 5.1 质检问题类型字典

| 编码 | 问题类型 | 说明 |
|---|---|---|
| STAFF_RESPONSE_SLOW | 响应慢 | 工作时间客户有效问题后超时回复 |
| STAFF_IGNORE_QUESTION | 忽略问题 | 未回答客户核心问题 |
| STAFF_ATTITUDE_COLD | 态度冷淡 | 敷衍、不耐烦、机械回复 |
| STAFF_ATTITUDE_CONFLICT | 情绪冲突 | 争吵、讽刺、攻击客户 |
| STAFF_NEED_PROBE_MISSING | 未追问需求 | 未确认规格、数量、场景、预算 |
| STAFF_PRODUCT_UNPROFESSIONAL | 商品不专业 | 规格、材质、承重、物流解释不足 |
| STAFF_QUOTE_PREMATURE | 过早报价 | 信息不足直接报价 |
| STAFF_QUOTE_MISSING | 应报价未报价 | 客户高意向但没有报价或区间 |
| STAFF_CONVERSION_WEAK | 推进不足 | 有意向但未引导下单、留资或下一步 |
| STAFF_CONTACT_MISSING | 应留资未留资 | 定制、批量、高意向客户未询问联系方式 |
| STAFF_CONTACT_POOR | 留资方式差 | 强迫式、重复式、没有服务前置 |
| STAFF_HANDOFF_MISSING | 留资后未交接 | 客户给联系方式后未确认、未记录 |
| STAFF_FOLLOWUP_MISSING | 沉默未跟进 | 高意向客户沉默后无跟进 |
| STAFF_FALSE_PROMISE | 违规承诺 | 绝对化承诺效果、时效、质量 |
| STAFF_PRIVACY_RISK | 隐私风险 | 索要无关敏感信息 |

## 6. 客户意向度维度

客户意向度满分 100 分。该分数评价客户，不评价客服。

| 编码 | 维度 | 权重 | 高意向信号 | 低意向/风险信号 | 客服应对动作 | BI 字段 |
|---|---|---:|---|---|---|---|
| CI01 | 需求明确度 | 18 | 明确要买、定制、某规格、某数量 | 只说看看、随便问问 | 复述需求并推进下一步 | `need_type`、`need_summary` |
| CI02 | 场景与用途清晰度 | 10 | 阳台、露台、庭院、工程、店铺、室内、装修 | 场景不明 | 追问使用场景 | `usage_scene` |
| CI03 | 规格数量完整度 | 12 | 尺寸、数量、材质、承重、图片、图纸 | 关键参数缺失 | 补问尺寸/数量/材质 | `size_info`、`quantity_info`、`missing_infos` |
| CI04 | 预算与价格接受度 | 15 | 明确预算、接受报价、报价后继续问 | 超预算、只砍价、价格敏感 | 探预算、解释价值、给替代方案 | `budget_status`、`price_sensitivity` |
| CI05 | 时间紧迫度 | 10 | 今天、明天、本周、装修、开业、工期 | 年底、以后、先看看 | 优先确认库存/发货/工期 | `urgency_level`、`deadline_at` |
| CI06 | 互动投入度 | 10 | 多轮追问、主动发图、主动回复、问细节 | 单字回复、问完消失 | 继续深挖并推进 | `customer_message_count`、`question_count` |
| CI07 | 信任与品牌认可 | 8 | 看过评价、朋友推荐、认可质量/款式 | 明确不信任且拒绝解释 | 给案例、售后、材质说明 | `trust_signal` |
| CI08 | 联系方式/微信承接意愿 | 10 | 主动留微信/电话、同意加微信 | 拒绝沟通并离开 | 自然留资承接 | `contact_requested`、`contact_provided` |
| CI09 | 成交动作信号 | 7 | 问怎么拍、发货、运费、地址、发票、付款 | 明确不买 | 给下单路径、确认备注 | `order_signal`、`order_guided` |

### 6.1 意向等级

| 等级 | 条件建议 | 跟进动作 |
|---|---|---|
| H1 立即跟进 | 意向分 >= 85，且需求明确，价格/数量/时间/联系方式/下单信号至少命中 2 项 | 立即处理，优先报价、留资、下单引导 |
| H2 重点跟进 | 意向分 70-84，需求明确且持续互动 | 当天跟进，补齐缺失信息 |
| H3 常规跟进 | 意向分 50-69，有需求但预算/规格/联系方式未确认 | 每周 1-2 次，先补信息 |
| H4 待观察 | 意向分 30-49，泛了解、时间远、价格敏感 | 低频培育 |
| L 低意向 | 意向分 < 30，或明确拒绝/无购买需求 | 不重点跟进 |

### 6.2 高意向原因字典

| 原因 | 强信号示例 | 客服应对 |
|---|---|---|
| 明确商品需求 | “这个花箱能定制吗？” | 追问尺寸、场景、数量 |
| 明确规格尺寸 | “我要 3 米长的” | 确认高度、宽度、材质 |
| 明确数量/批量 | “需要 20 个左右” | 询问批量报价、发货地、联系方式 |
| 明确使用场景 | “露台要做一圈花槽” | 推荐适配方案，争取现场图 |
| 询问价格/报价 | “这个多少钱？” | 先补齐规格，再报价或给区间 |
| 报价后继续问 | “那多久能发？” | 引导下单或留联系方式 |
| 有时间节点 | “这周能发吗？” | 优先跟进，确认库存/工期 |
| 主动发图/尺寸 | 发现场图、参考图、图纸 | 进入方案沟通，建议微信承接 |
| 主动留联系方式 | 发微信/电话 | 确认收到，标记先发承接 |
| 问下单/物流 | “拍哪个链接？” | 给下单路径和备注说明 |

### 6.3 客户意向减分项

| 编码 | 信号 | 扣分建议 | 说明 |
|---|---|---:|---|
| CD01 | 明确拒绝 | -30 | 不需要、不考虑、已买别家 |
| CD02 | 预算明显不匹配 | -10 到 -25 | 需求与预算差距大 |
| CD03 | 高度价格敏感 | -8 到 -15 | 只砍价、只找最低价 |
| CD04 | 沉默流失 | -8 到 -20 | 48 小时以上无回复 |
| CD05 | 时间节点过远 | -5 到 -10 | 3 个月以后再说 |
| CD06 | 信息虚假/异常 | -20 到 -35 | 疑似广告、无效咨询 |
| CD07 | 信任障碍强 | -5 到 -15 | 明确不信任且拒绝解释 |
| CD08 | 重复无效咨询 | -5 到 -15 | 多次咨询但从不推进 |

## 8. 老板端与客服端统一开发表

下表是页面和 BI 开发时最重要的字段清单。客服端不能只显示意向标签，必须显示具体客户、原因、证据、缺失信息、下一步动作和建议话术。

| 序号 | 分组 | 维度/字段 | 老板端重点看什么 | 客服端重点看什么 | 客户明细必须展示 | 判断/证据口径 | 客服改进与下次话术 | BI/落库字段建议 |
|---:|---|---|---|---|---|---|---|---|
| 1 | 总览 | 高意向客户明细 | 全公司 H1/H2 客户有多少、谁在跟、有没有漏承接 | 我名下具体哪些客户有意向，今天先跟谁 | 客户账号、脱敏名、商品、意向等级、最后互动、负责客服 | 客户需求、报价、数量、时间、联系方式等证据消息 | 优先处理 H1/H2，不只看数量，要逐个跟进 | `customer_id`、`staff_id`、`intent_tier`、`intent_score` |
| 2 | 客户身份 | 客户是谁 | 哪些客服手里有高价值客户 | 这个客户是谁、之前有没有聊过 | 淘宝/旺旺账号、脱敏买家名、首次咨询时间、最近咨询时间 | `dim_customer` + 非客服 `chat_target` | 老客户先看历史需求，不要重复问已说过的信息 | `customer_account`、`buyer_wangwang_masked`、`first_seen_at`、`last_seen_at` |
| 3 | 意向等级 | H1/H2/H3/H4/L | 高意向客户池规模、客服是否接住机会 | 每个客户优先级，不要只显示“有意向” | 意向等级、意向分、等级原因 | AI 意向分 + 规则命中证据 | H1 立即跟，H2 当天跟，H3 补信息，H4 低频培育 | `intent_tier`、`intent_score`、`intent_reason_codes` |
| 4 | 意向原因 | 为什么有意向 | 高意向来自哪些信号：定制、批量、价格、时间、留资 | 这个客户为什么值得跟 | 命中的意向原因列表、每条原因对应聊天证据 | 每个原因必须绑定 `message_id` | 根据原因推进：问尺寸、报价、要联系方式、引导下单 | `intent_reasons`、`intent_evidence_message_ids` |
| 5 | 需求摘要 | 客户想要什么 | 团队主要需求类型分布 | 不用翻全聊天就知道客户要什么 | 需求一句话摘要、需求类型 | AI 从客户消息抽取，必须有证据 | 先复述需求再推进：`您这边主要是想做...对吗？` | `need_summary`、`need_type` |
| 6 | 商品信息 | 咨询哪个商品 | 哪些商品带来高意向，哪些商品问题多 | 客户具体咨询哪个商品 | 商品名、商品 ID、店铺、会话状态 | `dim_product`、`dim_shop`、`dwd_qn_conversation` | 围绕商品规格、价格、使用场景给建议 | `product_id`、`product_name`、`shop_id` |
| 7 | 使用场景 | 客户用在哪里 | 哪些场景转化强：阳台、庭院、工程、店铺 | 客户实际使用场景是什么 | 阳台/露台/庭院/工程/店铺/室内等 | 客户消息中场景词、图片、描述 | `您是阳台还是庭院使用？我按场景给您推荐合适规格。` | `usage_scene`、`scene_evidence_ids` |
| 8 | 规格尺寸 | 尺寸是否明确 | 定制类客户信息完整度 | 还缺不缺尺寸、长宽高、口径 | 已知尺寸、缺失尺寸、客户发图情况 | 尺寸数字、单位、图片、图纸消息 | `方便发一下大概长宽高吗？我按尺寸帮您确认能做的方案。` | `size_info`、`missing_size_fields` |
| 9 | 数量/批量 | 是否批量需求 | 批量/工程机会有多少 | 客户要几个、多少米、多少袋 | 数量、批量标记、工程标记 | 数量词、批发/工程/多件等关键词 | `大概需要几件/多少米？数量不同报价会不一样，我帮您核一下。` | `quantity_info`、`is_bulk_lead` |
| 10 | 预算状态 | 预算是否明确 | 高意向未确认预算的比例 | 要不要补问预算 | 明确预算、未确认、价格敏感、预算不匹配 | 预算金额、接受报价、嫌贵、砍价等消息 | `您大概预算在哪个范围？我可以按预算给您推荐合适方案。` | `budget_status`、`budget_amount`、`price_sensitivity` |
| 11 | 时间节点 | 是否急用 | 急单客户是否被及时承接 | 客户什么时候要 | 今天/明天/本周/装修/开业/工期/远期 | 日期、急用、工期、活动等关键词 | `您这边大概什么时候需要用到？我先帮您确认发货和工期。` | `urgency_level`、`deadline_at` |
| 12 | 互动投入 | 客户聊得深不深 | 流量质量和真实意向 | 客户是否主动发问、追问、发图 | 主动咨询、多轮追问、发图、问细节 | 客户消息轮次、问题数、图片/链接/尺寸 | 多轮客户要主动推进，不要只被动答 | `customer_message_count`、`question_count`、`asset_sent` |
| 13 | 价格反应 | 报价后反应 | 价格异议比例、商品价格接受度 | 客户是接受、犹豫还是拒绝 | 报价后继续问、嫌贵、砍价、沉默 | 报价消息后客户下一轮回复 | `我理解您关注价格，这款主要差在材质/尺寸；如果想控制预算，我给您另一个方案。` | `quote_reaction`、`price_objection` |
| 14 | 下单信号 | 是否接近成交 | 成交前信号是否被客服接住 | 客户是否问拍哪个、怎么发、运费、地址 | 下单方式、发货、运费、地址、付款状态 | 问下单/物流/地址/发票等消息，千牛状态 | `这款您可以拍这个链接，备注尺寸；我这边也帮您确认发货时间。` | `order_signal`、`qn_status` |
| 15 | 联系方式询问 | 客服有没有要联系方式 | 高意向客户留资询问率 | 我有没有对该客户询问微信/电话 | 是否询问、询问时间、询问话术 | 客服消息包含微信/电话/手机号/联系方式 | 合规正向动作：`千牛沟通方案不太方便，您方便留个微信或电话吗？` | `contact_requested`、`contact_request_message_id` |
| 16 | 联系方式提供 | 客户有没有给联系方式 | 留资成功率、线索质量 | 客户是否已经给微信/电话 | 是否提供、联系方式类型、脱敏值 | 客户消息中微信号/手机号/电话，需脱敏展示 | `收到，我记录一下，稍后把方案和报价同步给您。` | `contact_provided`、`contact_type`、`contact_value_masked` |
| 17 | 先发承接 | 有没有进入微信承接 | 留资到先发链路哪里断 | 我的客户是否已经交接到先发 | none/asked/provided/ready/matched/converted | 先发映射表或人工标记 | 客户给联系方式后必须确认并标记承接，不要断线 | `xianfa_handoff_status`、`xianfa_customer_id` |
| 18 | 留资风险 | 高意向未留资 | 哪些客服漏了高价值客户 | 哪些客户我应该要联系方式但没要 | H1/H2、定制/批量/复杂需求、未询问联系方式 | 意向等级 + 需求类型 + `contact_requested=false` | `这个需求需要确认方案，方便留个微信吗？我整理好发您。` | `high_intent_contact_missing` |
| 19 | 沉默风险 | 高意向客户沉默 | 高意向流失风险 | 哪些客户多久没回了 | 最后客户消息、最后客服消息、沉默小时数 | 时间差计算 | `亲，您上次咨询的尺寸我这边可以帮您继续确认，需要的话我给您整理个方案。` | `last_customer_message_at`、`silent_hours` |
| 20 | 下一步动作 | 客服现在该干什么 | 团队执行是否闭环 | 每个客户下一步：补需求/报价/留资/激活/下单 | 下一步动作、截止时间、完成状态 | AI 根据缺失信息和阶段生成 | 按动作执行，不要只看分数 | `next_action`、`next_action_deadline`、`action_status` |
| 21 | 建议话术 | 下次怎么说 | 话术质量是否可复制训练 | 直接可复制给客户的下一句 | 建议回复、适用原因、禁用话术 | 基于客户当前阶段和缺失信息生成 | 话术要具体，不要空泛：尺寸、数量、预算、联系方式必须带上 | `suggested_reply`、`reply_scene` |
| 22 | 客户阶段 | 漏斗位置 | 咨询到留资/下单转化漏斗 | 客户现在在哪一步 | 新咨询、需求明确、方案沟通、报价、留资、先发、下单、沉默、流失 | 会话内容 + 千牛状态 + 联系方式状态 | 阶段不同动作不同：报价阶段推链接，方案阶段补参数 | `lifecycle_stage` |
| 23 | 流失原因 | 为什么没继续 | 流失主要卡点 | 我的客户为什么掉了 | 价格、无货、物流、质量疑虑、回复慢、已买别家、未留资 | 客户拒绝消息、沉默、客服问题 | 针对原因挽回：价格给替代，信任给案例，尺寸给方案 | `lost_reason`、`lost_risk_level` |
| 24 | 响应及时 | 客服回得快不快 | 哪些客服响应慢导致机会损失 | 我哪个客户等太久了 | 首响、平均响应、超时次数 | 客户有效问题到客服回复的时间差 | 先安抚：`我先帮您确认一下，稍等马上回复您。` | `first_response_seconds`、`avg_response_seconds`、`timeout_count` |
| 25 | 问题解决 | 有没有答到点上 | 团队问题解决率 | 我漏答了客户哪个问题 | 客户问题、客服回答、是否解决 | 客户问句与后续客服回复匹配 | 先回答核心问题，再引导下一步 | `question_answer_rate`、`unanswered_questions` |
| 26 | 需求追问 | 有没有问全 | 客服是否会挖需求 | 我还缺哪些关键问题没问 | 缺尺寸、缺数量、缺预算、缺场景、缺时间 | 客户需求类型 + 必填信息缺失 | `价格和尺寸/数量有关，我先确认下您的尺寸和数量再报价。` | `missing_infos`、`need_probe_missing` |
| 27 | 报价推进 | 该报价有没有报价 | 高意向客户是否推进到报价 | 我是否该给客户报价/区间 | 已报价、未报价、过早报价、报价后反应 | 报价词、金额、价格区间、报价前信息完整度 | 信息不足先补问，信息够了就明确报价或给区间 | `quote_given`、`quote_missing`、`quote_premature` |
| 28 | 下单引导 | 有没有推进成交 | 客服成交推进能力 | 客户问下单后我有没有接住 | 链接、拍法、备注、发货、地址确认 | 下单信号后客服是否给明确路径 | `您拍这个规格就可以，备注尺寸；我这边帮您确认发货。` | `order_guided`、`conversion_weak` |
| 29 | 异议处理 | 贵/犹豫怎么处理 | 团队异议处理短板 | 我面对贵、再看看、不放心时怎么回 | 异议类型、客服回应、是否继续互动 | 客户异议消息 + 客服后续回应 | 先共情，再解释，再给选择 | `objection_type`、`objection_handled` |
| 30 | 态度温度 | 语气是否合适 | 服务体验风险 | 我的回复是否冷淡/敷衍 | 冷淡回复、冲突回复、礼貌称呼 | 负面词、短硬回复、冲突语气 | 加承接和解释：`可以的，我帮您看一下...` | `attitude_issue_count`、`sentiment_risk` |
| 31 | 合规风险 | 有没有硬风险 | 高风险会话和客服 | 我是否有虚假承诺/隐私过采 | 虚假承诺、绝对化表达、无关敏感信息 | 绝对化词、敏感信息索取、争吵证据 | 用条件式表达：`我先确认库存/工期后给您准确答复。` | `compliance_risk_count`、`risk_level` |
| 32 | 留资方式 | 留资问得好不好 | 留资成功率和骚扰风险 | 我是不是只要微信没服务 | 先服务后留资、强迫留资、重复留资 | 留资前是否回答核心问题，是否多次催促 | 先解决问题，再自然承接；不要只说“加微信” | `contact_quality`、`contact_poor_flag` |
| 33 | 留资后交接 | 给了联系方式后有没有接住 | 已留资未承接断点 | 客户给微信/电话后我有没有确认 | 已确认、未确认、已标记先发、未交接 | 客户提供联系方式后客服下一条消息和承接状态 | `收到，我记录一下，稍后把方案和报价同步给您。` | `handoff_missing`、`handoff_confirmed` |
| 34 | 客服不足 | 哪里要改 | 每个客服 TOP 问题和培训重点 | 我的扣分项、证据、原因 | 问题标题、扣分、证据消息、影响说明 | `qc_issue` + `qc_issue_evidence` | 每个问题给“正确做法 + 下次话术” | `issue_count`、`top_issue_types` |
| 35 | 改进建议 | 怎么改 | 团队培训内容 | 我下一次遇到同类客户怎么说 | 错误表现、客户影响、正确方法、推荐话术 | AI 基于问题证据生成 | 建议必须能复制使用，不能只写“加强沟通” | `suggested_action`、`suggested_reply` |
| 36 | 客服排行 | 谁接得好/差 | 平均分、H 客户承接率、留资率、问题数 | 客服端不看全局排行，只看自己 | 客服名、会话数、服务分、H 客户数、留资成功率 | 聚合 `qc_result` + 意向宽表 | 老板用于培训和管理，不给普通客服看别人数据 | `staff_daily_metrics` |
| 37 | 高意向承接率 | 机会有没有被接住 | 哪些客服漏机会 | 我名下 H 客户是否报价/留资/下单 | H 客户总数、已承接数、未承接明细 | H1/H2 + 报价/下单/联系方式状态 | H 客户不能停在“已回复”，必须有下一步 | `h_customer_count`、`h_customer_handoff_rate` |
| 38 | 商品机会 | 哪些商品机会多 | 商品高意向排行、价格异议、定制/批量需求 | 我接待该商品时常见客户问题 | 商品、H 客户数、需求类型、问题类型 | 商品维度聚合 | 针对高频问题准备标准回答 | `product_intent_metrics` |
| 39 | 店铺表现 | 店铺整体情况 | 店铺服务分、意向客户、承接率 | 客服只看自己相关店铺会话 | 店铺、客服、会话、问题、意向 | 店铺维度聚合 | 店铺差异用于老板管理，不增加客服负担 | `shop_metrics` |
| 40 | 证据追溯 | 判断从哪里来 | 所有结论可审计 | 我能点开看到原话 | 证据消息、说话方、时间、上下文 | 每个扣分/高意向/留资判断必须有消息 ID | 无证据不扣分；无证据不判强意向 | `evidence_message_ids`、`message_id` |

## 9. 客服端页面开发要求

客服端不是老板端的缩小版。客服登录后要围绕“我今天该跟哪些客户、我哪里要改、下次怎么说”来设计。

| 页面 | 目标 | 核心内容 | 必须支持 |
|---|---|---|---|
| 我的工作台 | 今天先处理什么 | 今日待跟进客户、H1/H2 客户、沉默高意向、未留资客户、最近扣分问题 | 点击进入客户明细 |
| 我的意向客户 | 看到具体哪些客户有意向 | 客户列表、意向等级、需求摘要、证据、联系方式状态、下一步建议 | 按 H 等级、沉默、留资状态排序 |
| 意向客户详情 | 理解客户为什么有意向 | 客户画像、商品、需求、预算、时间、聊天证据、历史会话、建议跟进 | 展示证据消息和建议话术 |
| 我的质检问题 | 知道自己哪里不足 | 响应、解决、追问、报价、留资、态度、合规等问题聚合 | 可跳转原会话证据 |
| 我的改进建议 | 下次应该怎么说 | 问题原因、错误示例、正确做法、推荐话术 | 建议话术可复制 |
| 我的跟进任务 | 把客户推进下去 | 待补需求、待报价、待留资、待承接、待激活 | 可标记已处理 |

### 9.1 我的意向客户列表字段

| 字段 | 示例 | 说明 |
|---|---|---|
| 客户 | `aizgwp / a**` | 真实淘宝/旺旺账号 + 脱敏别名 |
| 意向等级 | `H1 / H2 / H3 / H4 / L` | 跟进优先级 |
| 意向分 | `86` | 客户购买/留资/转化可能性 |
| 当前阶段 | `方案沟通 / 报价阶段 / 留资承接` | 漏斗位置 |
| 需求摘要 | `阳台花箱定制，询问 3 米尺寸和材质` | 不用翻聊天 |
| 咨询商品 | `不锈钢异形花箱` | 商品机会 |
| 关键意向原因 | `明确尺寸 + 问报价 + 愿意加微信` | 解释为什么有意向 |
| 缺失信息 | `未确认数量、预算` | 下一步补什么 |
| 联系方式状态 | `未询问 / 已询问 / 已提供 / 已进先发` | 留资承接进度 |
| 最后互动 | `2026-06-24 21:33` | 最近客户/客服消息 |
| 沉默时长 | `18 小时` | 高意向沉默提醒 |
| 下一步动作 | `补问尺寸并争取微信` | 具体行动 |
| 建议话术 | `您方便发下现场尺寸吗...` | 可复制 |
| 风险提醒 | `高意向未留资` | 机会损失提醒 |
| 证据入口 | `3 条证据` | 点开看原始聊天 |

### 9.2 客服待办任务

| 待办类型 | 触发条件 | 展示文案 | 完成标准 |
|---|---|---|---|
| 待补需求 | H1/H2 客户缺尺寸、数量、场景、预算 | 补问客户尺寸/数量 | 客户回复关键需求信息 |
| 待报价 | 客户需求信息已够但未报价 | 给客户报价或价格区间 | 已发送报价消息 |
| 待留资 | H1/H2 客户未询问联系方式 | 争取微信/电话承接 | 已询问联系方式 |
| 待承接 | 客户已提供联系方式但未标记承接 | 确认联系方式并进入先发 | 标记 `ready/matched` |
| 待激活 | H1/H2 客户超过 24 小时未回复 | 发送跟进话术 | 客户回复或标记暂缓 |
| 待复盘 | 会话出现中高风险扣分 | 查看问题并学习建议话术 | 客服已读或完成培训动作 |

## 10. 老板端页面开发要求

老板端要看管理全局：服务质量、商机质量、客服能力、机会损失。

| 页面 | 目标 | 核心内容 |
|---|---|---|
| 老板工作台 | 看全局健康度 | 总会话、质检均分、高意向客户数、留资成功率、高风险数 |
| 高意向客户池 | 看哪些客户值得重点追 | 全部 H1/H2 客户、负责客服、承接状态、风险 |
| 客服表现 | 看谁服务好、谁漏机会 | 平均分、H 客户数、留资询问率、推进不足数 |
| 商机漏斗 | 看从咨询到留资/下单的转化 | 咨询、需求明确、报价、留资、先发承接、下单付款 |
| 问题分布 | 看共性培训点 | 响应、追问、解决、报价、留资、合规 |
| 商品机会 | 看哪些商品带来高意向 | 商品咨询量、高意向数、价格异议率、定制/批量需求 |
| 风险会话 | 看需要管理介入的问题 | 高意向未承接、客户投诉、冲突、虚假承诺 |

### 10.1 老板端核心指标

| 指标 | 口径 | 老板看这个为了什么 |
|---|---|---|
| 已质检会话数 | `qc_result` 数量 | 判断覆盖率 |
| 平均服务分 | 客服服务分平均值 | 看整体服务质量 |
| H1/H2 客户数 | 高意向客户数量 | 看机会规模 |
| 高意向占比 | H1/H2 / 有效会话 | 看流量质量 |
| 高意向承接率 | H1/H2 中已报价、留资或下单的比例 | 看客服是否接住机会 |
| 留资询问率 | 询问联系方式会话 / 有效会话 | 看获客动作是否执行 |
| 留资成功率 | 客户提供联系方式 / 已询问联系方式 | 看承接话术效果 |
| 先发承接率 | 已进入先发 / 已提供联系方式 | 看线索是否断点 |

| 高风险问题数 | high/critical 问题数 | 看服务风险 |
| 客服问题 TOP | 各客服最多的问题类型 | 定培训重点 |
| 商品高意向 TOP | H1/H2 最多的商品 | 调商品和投放策略 |

## 11. 先发微信承接规则

| 状态 | 说明 |

## 14. 证据规则

| 输出 | 是否必须有证据 | 说明 |
|---|---|---|
| 客服扣分问题 | 必须 | 对应 `qc_issue_evidence` |
| 客户 H1/H2 判断 | 必须 | 至少 1 条客户消息证据 |
| 意向原因 | 必须 | 每条原因绑定消息 |
| 联系方式已询问 | 必须 | 绑定客服询问消息 |
| 联系方式已提供 | 必须 | 绑定客户提供消息，展示时脱敏 |
| 客户明确拒绝 | 必须 | 绑定拒绝消息 |
| 客户沉默 | 可由时间差计算 | 不需要消息证据，但要有时间字段 |
| 响应超时 | 可由时间差计算 | 不需要消息证据，但要有时间字段 |

无证据时只能输出为“疑似”或“不确定”，不得正式扣分，不得判强意向。

## 15. 权限规则

| 场景 | 老板端 | 客服端 |
|---|---|---|
| 意向客户列表 | 全部客户，可按客服筛选 | 只看自己负责客户 |
| 意向证据 | 可看全部会话证据 | 只看自己会话证据 |
| 客服不足 | 看所有客服问题与排行 | 只看自己的问题 |
| 建议话术 | 可查看和评估 | 可复制使用 |
| 先发承接状态 | 看全局断点 | 只看自己客户的承接状态 |
| 高意向未留资提醒 | 全局风险 | 我的待办 |
| 客服排行 | 可见 | 不可见 |
| 规则配置 | 可见 | 不可见 |

后端接口必须强制数据权限。客服账号必须绑定 `dim_staff.sys_user_id`，客服端所有查询默认加：

```text
dwd_qn_conversation.staff_id = 当前登录用户绑定的 dim_staff.staff_id
```

## 17. 页面验收标准

| 页面 | 验收标准 |

| 我的意向客户 | 能看到具体客户列表，而不是只有 H 客户数量 |
| 客户详情 | 能看到意向原因、证据消息、缺失信息、下一步动作 |
| 我的改进建议 | 每个问题都有原因、证据、改进方法和建议话术 |
| 我的待办 | 高意向未跟进、未留资、已留资未承接能形成任务 |

| 客服表现 | 老板能看到客服接住高意向客户的能力，而不只是平均分 |

| 权限 | 客服无法查看其他客服客户和问题 |
| 证据 | 每个扣分和强意向判断都能点开原始消息 |

## 18. 最终开发口径

客服真正需要的是：

1. 哪些客户有意向；
2. 这些客户为什么有意向；
3. 这些客户现在缺什么信息；
4. 我有没有漏掉报价、留资、承接、跟进；
5. 我哪里做得不好；
6. 下次我应该怎么问、怎么说、怎么推进。

老板真正需要的是：

1. 哪些高意向客户有没有被接住；
2. 哪些客服经常漏高意向客户；
3. 留资和先发承接链路哪里断；
4. 哪些商品带来更高质量机会；
5. 团队共性短板应该怎么培训。

本文件作为后续 SmartQA 质检、客户意向、BI 页面、AI Prompt、ADS 宽表开发的统一依据。
