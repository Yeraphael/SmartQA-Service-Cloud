# 会议讨论深度分析

> 分析依据：`docs/会议讨论图.jpg` 与本次业务补充说明
> 生成日期：2026-06-27
> 当前结论：本项目应从“泛客服质检后台”明确升级为“每日批处理的客服服务质量与客户商机分析系统”。系统不追求实时陪聊，而是通过 RPA 每日固定时间获取数据，夜间批量质检分析，第二天 8 点前把老板端和客服端结果呈现出来。

## 1. 本次会议后的关键纠偏

这次讨论最重要的纠偏有两个。

第一，时效不是实时。

当前拿不到实时千牛数据，系统不应该设计成实时在线辅助客服。正确路径是：

```text
RPA 每天固定时间拉取千牛数据
  -> SmartQA 夜间同步和入库
  -> 批量 AI 质检和客户意向分析
  -> 生成老板端/客服端看板
  -> 第二天早上 8 点可查看
```

因此，“时效性”的产品目标不是秒级或分钟级提醒，而是每天稳定产出可用结果。只要老板和客服第二天 8 点能看到前一天完整分析，就符合当前阶段业务要求。

第二，留资逻辑必须区分标准化产品和定制化产品。

之前的分析容易把“留微信/电话/联系方式”理解成所有高意向客户都应该推动的动作，这是不准确的。这个公司的产品分为两类：

| 产品类型 | 正确销售动作 | 质检判断重点 |
|---|---|---|
| 标准化产品 | 直接解答、推荐、报价、推动下单 | 不应该无脑转微信，重点看是否讲清楚、是否推动成交 |
| 定制化产品 | 需要确认需求、尺寸、场景、数量等，再留资转微信/电话承接 | 应识别定制需求，并判断是否自然留资、是否承接 |

所以，“留资”不是通用加分项。只有在定制化、复杂报价、方案沟通、大件批量、需要后续承接等场景下，留资才是正确推进动作。对标准化商品，客服应该优先在千牛内完成服务和成交推进。

## 2. 项目重新定位

项目不应被定义成单纯的“AI 客服质检系统”，也不应被做成实时客服助手。更准确的定位是：

```text
SmartQA 每日客服服务质量与客户商机分析系统
```

它的核心价值是：

- 每天把千牛客服对话转成老板能看的经营面板。
- 每天把客服自己的问题和可跟进客户推送给客服。
- 判断客服服务质量、专业度、态度、敏感风险。
- 判断客户意向、商品机会、标准化/定制化路径是否正确。
- 让老板知道“谁做得好、谁有风险、哪些商品有机会、哪些会话值得复盘”。

一句话概括：

```text
每天晚上分析，第二天早上让老板能管人、管服务、看商机，让客服知道自己该改哪里、该跟谁。
```

## 3. 数据时效设计

### 3.1 当前阶段不做实时

当前不做：

- 实时监听千牛聊天。
- 客服边聊边弹提示。
- 分钟级预警。
- 在线话术实时推荐。

原因很明确：当前获取不到实时数据，数据来自 RPA 每日固定时间获取。

### 3.2 当前阶段应做每日闭环

建议把每日任务设计为固定链路：

| 时间 | 动作 | 说明 |
|---|---|---|
| 每天晚上固定时间 | RPA 获取千牛数据 | 与 RPA 约定数据落地方式 |
| RPA 完成后 | SmartQA 同步源数据 | 去重入库，生成会话、消息、客户、客服、商品维表 |
| 夜间 | AI 批量质检与意向分析 | 优先覆盖所有客服，再逐步扩大到全量 |
| 次日 8 点前 | 生成老板端/客服端看板 | 老板看全局，客服看本人 |

### 3.3 老板端仍需支持手动触发

虽然主链路是每天跑批，但老板端可以保留：

- 立即同步数据。
- 立即创建分析任务。
- 查看最近一次同步状态。
- 查看最近一次 AI 分析完成情况。

这里的“立即”不是实时业务能力，而是管理动作，方便排查和补跑。

## 4. 四个分析对象的正确理解

会议图里的四个分析对象，可以继续沿用我们之前“客服维度 + 客户维度”的总体思路，但需要加入“产品类型和销售路径”的判断。

### 4.1 客服回复质量

客服回复质量关注的是客服有没有把客户的问题回答好。

核心维度包括：

- 响应是否及时。
- 是否正面回答客户问题。
- 是否理解客户需求。
- 是否解释商品规格、材质、价格、物流、售后等关键信息。
- 是否存在模板化、敷衍、冷淡、答非所问。
- 是否在客户有明确需求时继续推进。

这部分仍然是客服质检的基础。

### 4.2 客服服务专业度

服务专业度不仅是态度好不好，更重要的是客服是否懂产品、懂场景、懂销售路径。

特别要区分：

| 场景 | 专业做法 | 不专业做法 |
|---|---|---|
| 标准化产品咨询 | 直接说明规格、价格、适用场景，推动下单 | 不回答核心问题，直接要微信 |
| 定制化产品咨询 | 追问尺寸、数量、场景、预算、交期，再自然留资 | 没问需求就报价，或没承接就结束 |
| 客户比价 | 解释差异、材质、服务、交付优势 | 只说“我们质量好” |
| 客户问物流/安装 | 说明发货、配送、安装边界 | 回答含糊或忽略 |

服务专业度应该成为老板端排行的重要维度。

### 4.3 客户意向与产品属性

客户意向不能脱离产品属性判断。

同样一句“能加微信吗”，在不同场景下含义不同：

- 客户咨询定制方案，主动要求加微信，通常是高意向。
- 客服面对标准化商品，不回答问题直接要求客户加微信，可能是服务路径错误。
- 客户没有明确需求，客服反复索要联系方式，可能是体验问题。

因此，客户意向分析必须同时输出：

- 客户意向等级。
- 客户需求摘要。
- 客户关注的产品属性。
- 产品类型判断：标准化 / 定制化 / 不确定。
- 是否需要留资承接。
- 当前最合适的下一步动作。

### 4.4 客服销售路径是否正确

这是本次会议最关键的新增判断。

系统要判断客服有没有走对路径：

```text
客户问题
  -> 判断产品类型
  -> 标准化：解答 + 推荐 + 报价/链接 + 下单推进
  -> 定制化：追问需求 + 方案/估价 + 留资 + 转微信/电话承接
  -> 沉默或异议：跟进或异议处理
```

路径错误的典型情况：

- 标准化商品本可以直接成交，却过早要求客户加微信。
- 定制化商品需要追问尺寸/数量/场景，但客服没有追问。
- 客户高意向，但客服没有报价、没有下一步。
- 客户适合留资承接，但客服没有留资。
- 客户已留资，但客服没有确认或没有交接。
- 客户只是在随便问问，客服强推联系方式导致体验差。

## 5. 标准化产品与定制化产品的质检规则

### 5.1 标准化产品

标准化产品的目标是直接在千牛内完成销售推进。

客服应该做：

- 快速回答商品问题。
- 给清楚规格、价格、库存、物流、使用场景。
- 推荐合适商品。
- 引导拍下、下单或继续咨询。
- 处理价格、质量、物流等异议。

客服不应该做：

- 没回答问题就要求加微信。
- 把所有客户都往线下转。
- 用“加微信详聊”替代正常商品介绍。
- 对标准品不报价、不解释、不推下单。

对标准化产品来说，留资不是天然加分项。只有客户主动提出、订单复杂、需要特殊沟通时才合理。

### 5.2 定制化产品

定制化产品的目标是确认需求后完成留资承接。

客服应该做：

- 追问尺寸、数量、使用场景、预算、交期。
- 判断是否需要方案、报价、设计、定制沟通。
- 在服务充分之后自然引导联系方式。
- 说明留资原因，例如方便发方案、核价、确认细节。
- 客户留资后确认记录和后续动作。

客服不应该做：

- 未确认需求直接报价。
- 客户明显需要定制却不追问。
- 客户适合承接但不留资。
- 客户已留资但没有确认。
- 生硬、重复、强迫式索要联系方式。

### 5.3 产品类型不确定时

如果 AI 无法判断产品是标准化还是定制化，应输出“不确定”，并根据对话证据给出建议：

- 客户是否提到尺寸、定做、方案、批量、工程。
- 商品名称是否暗示定制属性。
- 客服是否提到可定制、需要核价、发方案。
- 是否需要人工配置商品类型字典。

后续建议建立商品属性字典，让老板或运营可以维护：

| 字段 | 说明 |
|---|---|
| 商品 ID | 来自千牛商品数据 |
| 商品名称 | 用于识别 |
| 产品类型 | 标准化 / 定制化 / 混合 / 未配置 |
| 是否需要留资 | 是 / 否 / 条件触发 |
| 留资触发条件 | 大件、批量、定制尺寸、方案报价等 |

## 6. 老板端看板方向

右侧草图是老板端看板，理解方向基本是：

```text
总体服务质量 + 高风险会话 + 多维客服排行 + 商品机会 + 敏感词/风险检测
```

老板端应该是一套多维度经营面板，而不是简单列表。

### 6.1 顶部总览

建议包含：

- 总体服务质量分。
- 已分析会话数。
- 高风险会话数。
- 高意向客户数。
- 标准化商品直接成交推进不足数。
- 定制化商品应留资未留资数。
- 敏感词命中次数。
- 客服平均响应表现。

### 6.2 高风险会话

高风险会话包括：

- 敏感词命中。
- 态度冲突。
- 虚假承诺。
- 强迫式索要联系方式。
- 客户明显不满。
- 高意向客户被错误处理。
- 定制客户留资后未承接。

老板需要能点击进入会话证据，而不是只看到数字。

### 6.3 客服多维排行

排行不能只有一个总分。建议至少包含：

| 排行维度 | 说明 |
|---|---|
| 综合服务质量 | 整体质检得分 |
| 服务态度 | 礼貌、耐心、情绪、冲突 |
| 服务专业度 | 商品解释、方案能力、问题解决 |
| 响应及时性 | 首响、平均响应、长时间未回复 |
| 销售路径正确率 | 标准化/定制化路径是否走对 |
| 高意向承接能力 | 高意向客户是否报价、留资、跟进 |
| 敏感风险次数 | 敏感词、违规承诺、隐私风险 |
| 模板化/敷衍率 | 是否大量复制话术、没有针对性 |

这部分应该是老板端的核心，因为老板最终要看人、管人、培训人。

### 6.4 商品机会面板

商品机会可以作为老板端增强面板。

建议看：

- 咨询量高的商品。
- 高意向客户集中的商品。
- 定制化需求多的商品。
- 标准化商品咨询多但推进弱的商品。
- 客服解释问题多的商品。
- 价格异议多的商品。
- 应留资未留资多的商品。

商品机会不能伪造成订单或 ROI，因为当前没有真实订单金额数据。它应该被定义为“咨询与意向机会”，不是成交业绩。

### 6.5 敏感词检测

敏感词检测应包含：

- 绝对化承诺。
- 虚假效果承诺。
- 隐私或联系方式不当索取。
- 冲突、辱骂、攻击性表达。
- 平台规则风险词。
- 其他公司自定义敏感词。

敏感词不应只做命中统计，还要关联会话、客服、消息证据。

## 7. 客服端方向

客服端应服务于“第二天改进和跟进”，不是实时弹窗。

客服端核心页面建议：

| 页面 | 作用 |
|---|---|
| 我的工作台 | 昨日服务概览、扣分点、待跟进客户 |
| 我的高风险会话 | 查看自己哪些会话有风险 |
| 我的待跟进客户 | 高意向、应报价、应留资、已留资待确认 |
| 我的质检明细 | 查看会话证据和扣分原因 |
| 我的改进建议 | 针对态度、专业度、路径问题给建议话术 |

客服端最重要的是把 AI 结果变成行动：

- 哪个客户要继续跟。
- 哪个标准化客户应该直接推动下单。
- 哪个定制化客户应该补留资。
- 哪个客户已经留资但需要确认承接。
- 自己昨天最常见的问题是什么。

## 9. 数据库与规则配置建议

建议后续增加或强化这些业务数据：

| 数据 | 用途 |
|---|---|
| 商品类型配置 | 区分标准化、定制化、混合型 |
| 会话产品路径结果 | 保存 AI 对标准/定制路径的判断 |
| 敏感词规则 | 老板可维护风险词 |
| 客服多维评分 | 态度、专业度、响应、路径、风险 |
| 待跟进客户池 | 客服端第二天可执行 |
| 高风险会话池 | 老板端重点查看 |
| 商品机会统计 | 老板端看产品机会 |

特别注意：商品类型最好不要完全依赖 AI 猜测。AI 可以先判断，但系统应该允许后续维护商品字典，逐步把标准化/定制化规则沉淀下来。

## 10. 下一版开发方向

下一版不应该继续单纯堆页面，而应围绕“每日结果可用”重构。

### 10.1 后端优先级

- 打通 RPA 每日数据落地后的同步入口。
- 完善每日批量任务：同步、扩表、AI 分析、结果汇总。
- 扩展 AI 输出：产品类型、销售路径、留资合理性、下一步动作。
- 建立商品类型配置表。
- 建立高风险会话、待跟进客户、商品机会的聚合接口。

### 10.2 前端优先级

- 老板端首页改成多维看板。
- 老板端增加高风险会话列表。
- 老板端增加客服多维排行。
- 老板端增加商品机会面板。
- 客服端首页改成昨日结果和待跟进工作台。
- 会话详情显示证据、路径判断和建议话术。

### 10.3 质检规则优先级

- 重写“留资”判断：区分标准化与定制化。
- 标准化产品重点看解答、推荐、报价、下单推进。
- 定制化产品重点看追问、方案、留资、承接。
- 敏感词和风险判断独立成维度。
- 客服态度、专业度、响应、路径正确性分别计分。

## 11. 验收标准

下一版验收建议围绕业务结果，而不是页面数量。

| 验收项 | 标准 |
|---|---|
| 每日可用 | 前一天数据能在第二天 8 点前完成同步、分析和展示 |
| 产品路径正确 | 标准化商品不会被误判为必须留资，定制化商品能识别留资承接需求 |
| 老板端有管理价值 | 能看到总体服务质量、高风险、客服排行、商品机会、敏感词 |
| 客服端有行动价值 | 能看到自己的问题、待跟进客户、建议话术 |
| 数据真实 | 所有指标来自真实千牛数据和真实 AI 分析 |
| 证据可追溯 | 所有风险、扣分、意向和路径判断都能点到聊天证据 |
| 不伪造指标 | 没有订单/成交金额/ROI 数据时，不展示假成交结果 |

## 12. 最终结论

本次会议后的准确方向是：

```text
每天固定批量分析千牛客服会话，
围绕客服服务质量、客户意向、产品标准化/定制化路径、高风险和商品机会，
给老板一个多维管理看板，
给客服一个第二天可执行的改进与跟进工作台。
```

下一步开发最关键的是修正“留资逻辑”和“产品路径逻辑”。
不是所有高意向都要转微信，也不是所有留资都是好服务。
标准化产品应优先直接成交，定制化产品才重点进入留资和微信承接链路。
