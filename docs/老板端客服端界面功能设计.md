# 老板端客服端界面功能设计

> 项目：SmartQA Service Cloud
> 定位：每日客服服务质量、客户机会、商品机会与会话证据分析系统
> 核心原则：工作台看摘要，客服表现看人，客户商机看客户，商品机会看商品，会话复盘看证据。

## 1. 产品目标

SmartQA 面向老板端和客服端两个角色。

老板端用于每日经营管理：快速看到团队服务质量、需要关注的客服、高风险会话、高意向未跟进、商品机会和趋势，并能进入证据页面查看原始会话。

客服端用于个人改进和客户跟进：客服打开系统后知道今天要跟进谁、为什么要跟进、下一步怎么说、自己昨天有哪些问题，以及对应会话证据。

系统以每日批量分析为核心链路：

```text
RPA 获取千牛数据
  -> SmartQA 同步入库
  -> AI 批量质检与商机识别
  -> 次日老板端和客服端展示结果
```

## 2. 信息架构

### 2.1 老板端侧边栏

```text
工作台总览
客服表现
客户商机
商品机会
会话复盘
客服管理
数据与配置
  AI分析任务
  每日数据批次
  规则配置
```

### 2.2 客服端侧边栏

```text
我的工作台
客户跟进
会话复盘
个人账号
```

### 2.3 页面对象归属

| 页面 | 主对象 | 页面职责 |
|---|---|---|
| 工作台总览 | 每日摘要 | 老板第一眼看到重点、异常、趋势和跳转入口 |
| 客服表现 | 客服 | 看客服多维表现、排名、问题、证据和建议话术 |
| 客户商机 | 客户 | 看高意向客户、未跟进客户、建议动作和负责人 |

| 会话复盘 | 会话 | 看聊天原文、AI总结、问题证据和建议话术 |
| 客服管理 | 账号 | 管理客服账号、千牛账号绑定、状态和密码 |
| 数据与配置 | 系统支撑 | 管理AI分析任务、每日数据批次、规则配置 |
| 我的工作台 | 个人摘要 | 客服看到本人今日重点、待跟进任务和改进点 |
| 客户跟进 | 本人客户 | 客服处理本人名下客户机会 |
| 个人账号 | 本人账号 | 查看身份绑定、登录账号和修改密码 |

## 3. 老板端工作台总览

### 3.1 页面目标

老板打开系统后，第一屏回答三个问题：

```text
今天团队整体服务质量如何
哪些客服需要重点关注
客服多维表现排名发生了什么变化
商品机会和近 7 日趋势如何
```

工作台总览展示团队服务质量、客服质检榜单、选中客服详情、客服能力四象限、商品机会和近 7 天趋势。

工作台采用统一大屏排版：顶部状态与指标在上方，左侧为可滚动客服质检榜单，中间为选中客服六边形详情，右侧为客服能力四象限，底部横向展开商品机会 Top5 与近 7 天趋势。左侧菜单展开、收起以及进入全屏状态时，模块位置和信息层级保持一致。

### 3.2 顶部数据状态条

顶部状态条展示数据可用性和分析完成情况，内容固定为以下四项。

| 字段 | 说明 |
|---|---|
| RPA获取时间 | 千牛数据完成获取的时间 |
| AI分析完成时间 | AI质检与客户意向分析完成时间 |
| 已分析会话数 | 系统当前已分析的全部会话数量 |
| 系统状态 | 正常、处理中、部分失败、待处理 |

页面使用系统全屏入口进入全屏模式。

### 3.3 顶部核心指标

顶部核心指标卡片展示当前值。

| 指标 | 说明 |
|---|---|
| 总体服务质量分 | 团队当日服务健康度 |
| 高风险会话 | 需要复盘或介入的会话数量 |
| 需关注客服数 | 当日服务表现或风险表现需要老板关注的客服数量 |
| 高意向未跟进数 | 有购买机会但跟进动作不足的客户数量 |

### 3.4 客服质检榜单

客服质检榜单是工作台总览的核心区域，占据页面最大视觉面积。榜单支持纵向滚动，老板触碰到哪一行，该行高亮；点击客服后，右侧客服详情同步更新。

维度切换项：

| 维度 | 关注内容 |
|---|---|
| 综合表现 | 当前客服整体服务质量 |
| 响应效率 | 首响、平均响应、超时、长时间冷场 |
| 服务态度 | 礼貌、耐心、情绪、冲突处理、安抚能力 |
| 专业能力 | 商品、规格、材质、物流、安装、定制知识 |
| 问题解决 | 是否答到点上、是否漏答、是否解决核心疑问 |
| 需求挖掘 | 是否追问尺寸、数量、场景、预算、交期 |
| 成交推进 | 是否报价、给链接、引导下单、留资、跟进下一步 |

维度切换规则：

| 交互 | 效果 |
|---|---|
| 点击维度 | 当前维度高亮 |
| 切换维度 | 表格按当前维度得分重新排序 |
| 点击客服 | 右侧或中间展示当前客服详情 |
| 点击问题 | 跳转到该客服相关会话证据 |

客服表现总览表格字段：

| 字段 | 说明 |
|---|---|
| 排名 | 当前维度下的客服排名 |
| 客服 | 头像、姓名、客服组或千牛账号 |
| 当前维度得分 | 当前选中维度的得分 |
| 综合得分 | 该客服综合服务质量分 |
| 较综合排名 | 当前维度排名相对综合排名的差异 |
| 主要问题 | 当前维度下最突出的具体问题 |
| 状态 | 优秀、良好、需关注、需改进 |

榜单上方展示各维度第一名小卡片：

| 小卡片 | 说明 |
|---|---|
| 综合表现最佳 | 综合表现得分最高客服 |
| 响应效率最佳 | 响应效率得分最高客服 |
| 服务态度最佳 | 服务态度得分最高客服 |
| 专业能力最佳 | 专业能力得分最高客服 |
| 问题解决最佳 | 问题解决得分最高客服 |
| 需求挖掘最佳 | 需求挖掘得分最高客服 |
| 成交推进最佳 | 成交推进得分最高客服 |

视觉表达：

| 分数区间 | 表达 |
|---|---|
| 90-100 | 深绿，表现优秀 |
| 80-89 | 浅绿，表现良好 |
| 70-79 | 黄色，轻微关注 |
| 60-69 | 橙色，需要关注 |
| 0-59 | 红色，重点关注 |

### 3.5 选中客服详情

点击客服后展示当前客服详情。

| 内容 | 说明 |
|---|---|
| 客服信息 | 头像、姓名、认证/绑定状态、客服角色 |
| 综合服务分 | 当前客服综合服务质量分 |
| 六边形能力图 | 响应效率、服务态度、专业能力、问题解决、需求挖掘、成交推进 |
| 高风险会话数 | 当前客服相关高风险会话数量 |
| 待跟进意向数 | 当前客服名下需要继续跟进的意向客户数量 |

选中客服详情作为中间独立面板展示。

### 3.6 客服能力四象限

客服能力四象限展示团队客服在服务质量和服务效率两个方向上的位置。

| 字段 | 说明 |
|---|---|
| 横轴 | 服务效率分 |
| 纵轴 | 服务质量分 |
| 点位 | 每个客服一个点 |
| 高亮 | 当前选中客服点位高亮 |

点击四象限里的客服点位，客服详情同步更新。

### 3.7 商品机会 Top 5

底部展示商品机会 Top 5。

| 字段 | 说明 |
|---|---|
| 排名 | 当前商品机会排名 |
| 商品/功能 | 商品或功能名称 |
| 相关会话数 | 当前周期相关会话数量 |
| 高意向数 | 当前周期高意向客户数量 |
| 转化潜力 | 基于高意向率展示星级 |

### 3.8 近 7 天趋势

近 7 天趋势用于判断团队服务变化方向。

| 趋势 | 展示方式 |
|---|---|
| 质量分 | 折线图 |
| 高风险会话 | 柱线组合 |
| 高意向未跟进 | 折线图 |
| 会话量 | 折线图 |

### 3.9 统一大屏排版

工作台按照大屏结构固定排版，保证老板在菜单展开、菜单收起和全屏状态下看到一致的信息位置。

| 区域 | 排版 |
|---|---|
| 顶部 | 数据状态条 + 核心指标卡片 |
| 左侧 | 客服质检榜单大表，占页面最大宽度，支持滚动查看更多客服 |
| 中间 | 选中客服详情，使用六边形能力图展示六维得分 |
| 右侧 | 客服能力四象限 |
| 底部 | 商品机会 Top 5、近 7 天趋势横向展开 |

全屏状态用于老板开会、投屏和每日晨会查看。

## 4. 老板端客服表现

### 4.1 页面目标

客服表现页面用于管人、看问题和追证据。

老板在这里查看：

```text
谁表现好
谁需要关注
哪个维度弱
典型问题是什么
下一步怎么改
```

### 4.2 顶部筛选

| 筛选项 | 说明 |
|---|---|
| 日期 | 单日或时间范围 |
| 客服 | 单个或多个客服 |
| 客服组 | 按组筛选 |
| 维度 | 综合、响应效率、服务态度、专业能力、问题解决、需求挖掘、成交推进 |
| 风险等级 | 高、中、低 |
| 商品 | 按商品筛选相关会话 |

### 4.3 排名与矩阵

页面主体采用“维度切换 + 排名重排 + 详情抽屉”。

| 区域 | 内容 |
|---|---|
| 维度切换 | 综合与六个核心维度 |
| 客服排名 | 按当前维度排序 |
| 表现矩阵 | 客服 x 维度热力矩阵 |
| 详情抽屉 | 选中客服的分数、问题、证据、建议话术 |

### 4.4 客服详情

| 内容 | 说明 |
|---|---|
| 基础信息 | 姓名、账号、千牛绑定、客服组 |
| 综合分 | 当前周期综合服务分 |
| 六维分 | 六个维度拆分 |
| 趋势 | 近 7 日或近 30 日走势 |
| 问题 Top | 最突出问题类型 |
| 典型证据 | 关联会话入口 |
| 客户机会影响 | 该客服负责的未跟进、高意向客户 |
| 建议话术 | 可直接参考的话术建议 |

| 晚上 | 汇总全天数据，进入AI分析流程 |

### 9.3 规则配置

| 配置 | 说明 |
|---|---|
| 质检维度规则 | 六个核心维度的评分规则 |
| AI提示词 | 会话总结、问题识别、商机识别、话术建议提示词 |
| 风险表达规则 | 虚假承诺、情绪冲突、价格纠纷等表达规则 |
| 商品场景规则 | 不同商品、场景、定制需求的判断规则 |
| 建议话术库 | 客服可参考的标准话术 |
| 规则版本 | 记录规则版本、更新时间和启用状态 |

## 10. 客服端我的工作台

### 10.1 页面目标

客服打开系统后，第一屏知道今天要处理什么，以及自己需要改进什么。

```text
今天我要跟哪些客户
这些客户为什么重要
我昨天哪里做得不好
下一次应该怎么说
```

### 10.2 顶部摘要

4 个卡片：

| 指标 | 说明 |
|---|---|
| 待跟进客户 | 今日需要处理的客户数量 |
| 高风险会话 | 本人需要复盘或补救的会话数量 |
| 我的服务分 | 本人昨日服务质量分 |
| 主要改进点 | 本人最突出的一个问题 |

### 10.3 客户跟进队列

这是客服端最大模块。

| 字段 | 说明 |
|---|---|
| 优先级 | 高、中、低 |

| 商品 | 咨询商品 |
| 意向等级 | 高、中、低 |
| 跟进原因 | 为什么值得跟进 |

| 下一步动作 | 该做什么 |
| 建议话术 | 可参考话术 |
| 证据入口 | 跳转会话复盘 |

### 10.4 我的主要问题

| 内容 | 说明 |
|---|---|
| 问题类型 | 本人最突出的 1-3 个问题 |
| 影响会话数 | 涉及本人会话数量 |
| 典型证据 | 跳转本人会话复盘 |
| 正确做法 | 简短改进建议 |
| 推荐话术 | 下次可参考表达 |

### 10.5 最近复盘

| 内容 | 说明 |
|---|---|

## 15. BI视觉与交互要求

### 15.1 整体风格

| 要求 | 说明 |
|---|---|
| 产品感 | 页面以业务数据和可执行动作作为核心 |
| 清晰 | 摘要、排行、证据入口层级分明 |
| 舒适 | 保持留白，减少密集表格压迫感 |
| 专业 | 使用稳定的颜色、图表、分数和标签体系 |
| 真实 | 页面全部绑定真实业务数据、AI分析结果和会话证据 |

### 15.2 图表建议

| 场景 | 推荐呈现 |
|---|---|
| 客服多维表现 | 热力矩阵、维度切换、排名重排 |

| 问题类型 | 横向条形图 |
| 客户机会状态 | 分组列表、状态标签、优先级队列 |
| 商品机会 | 排名条、问题分布、异议标签 |
| 数据批次 | 状态表、时间轴、成功失败统计 |
