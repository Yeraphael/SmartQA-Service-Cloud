# 老板端客服端界面功能设计

> 项目：SmartQA Service Cloud
> 定位：每日客服服务质量、客户机会、商品机会与会话证据分析系统
> 核心体验：老板端看团队经营与管理动作，客服端看个人任务与改进动作，所有指标都能追溯到真实会话证据。

## 1. 产品定位

SmartQA Service Cloud 面向两个角色：

| 角色 | 使用目标 | 核心关注 |
|---|---|---|
| 老板 | 每日查看团队客服质量、风险、机会和趋势 | 谁表现好、谁需要关注、哪里有风险、哪里有机会 |
| 客服 | 每日处理本人客户跟进和服务改进 | 今天跟谁、为什么跟、怎么说、哪里需要改 |

系统以每日批量数据为核心链路：

```text
RPA 获取千牛数据
  -> SmartQA 数据同步入库
  -> AI 批量质检与机会识别
  -> 次日老板端与客服端展示分析结果
  -> 老板管理复盘，客服执行跟进
```

## 2. 统一视觉与交互基准

### 2.1 视觉风格

整体采用浅色企业 SaaS 管理后台风格。页面背景为极浅灰白，业务模块使用白色卡片承载，卡片之间通过浅蓝灰边框、轻阴影和留白区分层级。

| 类型 | 视觉规则 |
|---|---|
| 主色 | 蓝色，用于品牌、选中态、核心按钮、主指标 |
| 成功 | 绿色，用于正常、优秀、提升、完成 |
| 风险 | 红色，用于高风险、需改进、异常 |
| 提醒 | 橙色，用于需关注、机会、待跟进 |
| 时间 | 紫色，用于响应时长、处理耗时、时间类指标 |
| 卡片 | 白底、10-14px 圆角、浅蓝灰边框、轻阴影 |
| 图标 | 统一线性图标，KPI 图标可放大并带轻量背景 |
| 字体 | 标题清晰有力，表格紧凑可读，辅助信息用浅色小字 |

### 2.2 通用页面结构

| 区域 | 内容 |
|---|---|
| 左侧侧边栏 | 品牌 Logo、一级菜单、当前用户信息 |
| 顶部栏 | 页面标题、搜索、通知、全屏、主题、头像 |
| 数据区 | 筛选、状态、指标、图表、表格、证据入口 |
| 详情区 | 当前选中对象的画像、分数、问题、建议和证据 |

### 2.3 数据可信展示

所有经营页面展示数据批次和分析状态，让用户明确当前页面基于哪一天、哪一批、多少会话、多少客服的数据生成。

| 字段 | 说明 |
|---|---|
| 数据日期 | 当前分析结果对应的业务日期 |
| RPA 数据时间 | RPA 完成源数据获取的时间 |
| AI 分析完成时间 | AI 完成会话分析的时间 |
| 已分析会话数 | 当前批次完成 AI 分析的全部会话数量 |
| 覆盖客服数 | 当前批次覆盖到的客服数量 |
| 系统状态 | 正常、处理中、部分失败、待处理 |

## 3. 信息架构

### 3.1 老板端侧边栏

老板端侧边栏采用一级菜单结构，顺序如下：

```text
工作台总览
客服表现
客户商机
商品机会
会话复盘
规则配置
客服账号
数据与任务
```

| 菜单 | 页面职责 |
|---|---|
| 工作台总览 | 老板第一眼查看团队服务质量、风险、机会、排行和趋势 |
| 客服表现 | 按客服维度管理人、看排名、看短板、看证据 |
| 客户商机 | 管理高意向客户、待跟进客户和下一步动作 |
| 商品机会 | 从商品维度查看咨询热度、机会、问题和优化建议 |
| 会话复盘 | 查看完整聊天证据、AI 总结、问题原因和建议话术 |
| 规则配置 | 配置质检维度、AI 提示词、问题标签、建议动作和话术 |
| 客服账号 | 管理客服账号、千牛账号绑定、分组、启停和密码 |
| 数据与任务 | 查看数据批次、同步任务、AI 分析任务和重试状态 |

### 3.2 客服端侧边栏

客服端侧边栏聚焦个人执行，顺序如下：

```text
我的工作台
客户跟进
会话复盘
个人账号
```

| 菜单 | 页面职责 |
|---|---|
| 我的工作台 | 客服查看本人今日任务、服务分、问题和改进建议 |
| 客户跟进 | 客服处理本人名下客户机会和待跟进动作 |
| 会话复盘 | 客服查看本人会话证据、扣分原因和建议话术 |
| 个人账号 | 客服查看本人账号、千牛绑定、分组和密码设置 |

## 4. 老板端：工作台总览

### 4.1 页面定位

工作台总览是老板端首页，定位为管理驾驶舱。页面第一屏帮助老板快速判断：

| 问题 | 页面回答方式 |
|---|---|
| 今天整体服务质量怎么样 | 顶部服务质量分、近 7 天质量趋势 |
| 今天风险是否突出 | 高风险会话数、客服详情中的风险会话 |
| 机会是否被及时承接 | 高意向未跟进数、客服详情中的待跟进意向 |
| 哪些客服表现最好或最弱 | 客服质检维度卡片、客服排行榜 |
| 某个客服具体问题在哪里 | 客服详情、六边形能力图、改进建议 |
| 团队能力分布是否健康 | 客服能力四象限 |
| 哪些商品机会值得关注 | 商品机会 Top 5 |

### 4.2 整体排版

工作台采用“顶部状态 + KPI 指标 + 三列主分析区”的结构，侧边栏展开、侧边栏收起、全屏状态下保持同一套信息层级。

| 区域 | 内容 |
|---|---|
| 顶部栏 | 页面标题、全局工具图标、当前用户 |
| 数据同步信息栏 | 数据日期、RPA 数据时间、AI 分析完成时间、已分析会话数、覆盖客服数、系统状态 |
| 核心 KPI 区 | 总体服务质量分、高风险会话数、高意向未跟进数、平均响应时长、今日会话总数 |
| 左列 | 客服质检维度卡片、客服排行榜 |
| 中列 | 当前选中客服详情、商品机会 Top 5 |
| 右列 | 客服能力四象限、近 7 天趋势 |

主分析区比例建议：

| 列 | 占比 | 设计重点 |
|---|---:|---|
| 左列 | 42% | 最大区域，承载客服维度和排行榜 |
| 中列 | 24% | 展示当前客服详情和商品机会 |
| 右列 | 34% | 展示团队分布和趋势 |

### 4.3 侧边栏

侧边栏顶部展示 SmartQA Service Cloud 品牌标识。菜单项由图标和文字组成，当前页面“工作台总览”使用浅蓝底、蓝色文字和蓝色图标高亮。

侧边栏底部展示当前登录用户：

|---|---|
| 头像 | 当前老板头像 |
| 名称 | 老板 |
| 身份 | 超级管理员 |
| 状态 | 在线状态点 |

### 4.4 顶部栏

顶部栏左侧展示页面标题：

```text
老板工作台总览
```

顶部栏右侧展示系统级工具：

| KPI | 示例 | 颜色 | 业务含义 |

### 4.7 客服质检维度

客服质检维度是工作台核心区域的上半部分，使用一排可点击维度卡片呈现。每张维度卡片展示维度名、该维度第一名客服头像、客服姓名、得分和图标。

| 维度 | 关注内容 |
|---|---|
| 综合维度 | 客服整体服务质量和综合表现 |
| 响应效率 | 首响、平均响应、超时、冷场 |
| 服务态度 | 礼貌、耐心、情绪、安抚、冲突处理 |
| 专业能力 | 商品知识、规格、材质、物流、售后、定制理解 |
| 问题解决 | 是否答到点上、是否解决客户核心疑问 |
| 需求挖掘 | 是否追问尺寸、数量、场景、预算、交期 |
| 风险控制 | 是否识别纠纷、误导承诺、负面情绪和升级风险 |

维度卡片交互：

| 操作 | 页面反馈 |
|---|---|
| 点击维度卡片 | 当前维度高亮 |
| 切换维度 | 排行榜按当前维度得分重排 |
| 查看维度第一名 | 卡片内展示该维度当前最高分客服 |

### 4.8 客服排行榜

客服排行榜位于维度卡片下方，是老板管理客服的核心表格。表格支持纵向滚动和分页，鼠标悬停行高亮，点击行后同步刷新中间客服详情和右侧四象限高亮点位。

表格字段：

| 字段 | 说明 |
|---|---|
| 排名 | 当前维度下的排名，前三名使用金银铜奖牌图标 |
| 客服 | 头像、姓名、真实千牛账号 |
| 本维度得分 | 当前选中维度下的得分 |
| 综合得分 | 该客服综合服务质量分 |
| 排名变化 | 当前维度排名相对综合排名的差异 |
| 趋势 | 迷你折线图展示近几日变化 |
| 主要问题 | 当前最突出的 1 个问题标签 |
| 状态 | 优秀、良好、需关注、需改进 |
| 所属归属 | 优先展示真实客服组；未配置客服组时展示该客服会话最多的店铺；均无数据时显示未分组 |

分数和状态表达：

| 分数区间 | 状态 | 颜色 |

| 80-89 | 良好 | 浅绿 |
| 70-79 | 需关注 | 橙色 |
| 0-69 | 需改进 | 红色 |

## 6. 老板端：客户商机

### 6.1 页面目标

客户商机页面用于老板查看客户机会是否被及时承接，帮助管理高意向客户、未报价客户、待补问客户和沉默客户。

### 6.2 顶部统计

| 指标 | 说明 |
|---|---|
| 高意向客户数 | 当前周期 AI 识别出的高意向客户数量 |
| 待跟进客户数 | 需要客服继续处理的客户数量 |
| 未报价客户数 | 客户存在购买意向且需要报价的客户数量 |
| 未确认需求客户数 | 需要补问尺寸、数量、场景、预算、交期等信息的客户数量 |
| 沉默客户数 | 高意向后长时间无有效后续的客户数量 |

### 6.3 客户列表

| 字段 | 说明 |
|---|---|
